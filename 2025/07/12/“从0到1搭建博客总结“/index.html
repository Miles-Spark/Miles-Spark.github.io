“从0到1搭建博客总结“

一、搭建前的准备
搭建前需要下载这些软件：
Git：官网下载：https://git-scm.com/
Node.js 官网下载：http://nodejs.cn/

二、安装hexo
1.Hexo 是一款基于 Node.js 的快速、简洁且高效的静态博客框架。在安装好Node.js后进入cmd命令行 输入指令安装hexo
npm install -g hexo-cli
2.新建一个文件夹放hexo项目，然后右键打开git bash
[图片]
输入指令创建hexo项目，如果在创建过程中报错，可能是网络问题可以挂个梯子清空文件夹再创建一次。
hexo init
3.输入指令启动hexo项目
hexo s
[图片]
新建的项目中有：
node_modules: 依赖包
public：存放生成的页面
scaffolds：生成文章的一些模板
source：用来存放你的文章
themes：主题
** _config.yml: 博客的配置文件**

4.在本地尝试将网站打开
http://localhost:4000/
[图片]
出现这个页面说明就成功了。

三、将Hexo项目部署到Github上
将Hexo项目部署到GitHub上后就可以在多台设备上访问我们网站了。部署流程如下：
1.在GitHub上新建仓库
新建的仓库名必须是你的用户名.github.io，否则仓库名就是无效的无法部署成功。
[图片]

2.生成ssh添加到Github
ssh提供了安全验证和免密操作，hexo项目与github远端仓库交互时不需要每次输入用户名和密码。
第一步：配置git 中的用户名和邮箱，打开cmd, 输入以下命令
git config –global user.name “yourname”
git config –global user.email “youremail”
这里的yourname输入你的GitHub用户名，youremail输入你GitHub的邮箱。这样GitHub才能知道你是不是对应它的账户。用户名为仓库的名称，邮箱为注册github的邮箱

第二步： 在hexo项目的目录下 右键打开git bash 输入指令 生成公钥
ssh-keygen -t rsa -C 自己的邮箱地址
输入指令后 连敲击三次回车 出现下图就成功了
[图片]

第三步，创建的公钥在.ssh目录中，打开该目录 id_rsa.pub 表示公用复制这里面的内容，然后进入github.
点击setings
[图片]
[图片]
发现我们需要一个密钥，把我们刚刚复制的密钥粘进去，title随便起
[图片]

3.部署到github中
这一步，我们就可以将hexo和GitHub关联起来，也就是将hexo生成的文章部署到GitHub上。

第一步 修改配置文件
[图片]
修改内存
deploy:
type: git
repo: git@github.com:goubin18/goubin18.github.io.git //（用自己的仓库路径代替）
branch: main
[图片]

第二步安装 deploy-git
npm install hexo-deployer-git –save
[图片]

第三步 生成静态界面并推送到远端
hexo c #清除缓存文件 db.json 和已生成的静态文件 public
hexo g #生成网站静态文件到默认设置的 public 文件夹(hexo generate 的缩写)
hexo d #自动生成网站静态文件，并部署到设定的仓库(hexo deploy 的缩写)
注意deploy时会让输个yes

最后就可以使用xxx.github.io来访问你的博客啦

[图片]
